# llm.txt — AI Agent Brief

A dense brief for AI coding agents working in this repo. Read this first. For full detail, see `CONVENTIONS.md` (coding standards) and `README.md` (quick start).

## First Step: Detach From the Template Repo

**Before you touch anything else, run `bash detach.sh`.** The template is cloned from `erium/app-template`; leaving that remote attached risks accidentally pushing an app's work back upstream. The script is idempotent:

- If `origin` points at `erium/app-template`, it wipes `.git/` and runs `git init` + initial commit so your new project has a clean history.
- If there is no `.git/`, it initializes one.
- If `origin` is anything else (or missing), it exits without changes.

After detaching, add your own remote with `git remote add origin <your-repo-url>`. Do not push before running this script.

## What This Is

A generic full-stack TypeScript template: React frontend + Express REST backend + PostgreSQL via Drizzle ORM. Ships with auth, multi-tenancy (tenant → users → invitations), i18n (de/en), Stripe billing, PDF export (Playwright), and 53 shadcn/ui components. Single `package.json` (no monorepo). Branding lives in `shared/config.ts`.

## Stack

- **Runtime:** Node 20+, pnpm 10
- **Frontend:** React 19, Vite 7, wouter 3 (routing), TailwindCSS 4, shadcn/ui, react-hook-form 7, react-i18next 16, @tanstack/react-query 5, sonner (toasts)
- **Backend:** Express 4, drizzle-orm 0.44 + pg 8 (PostgreSQL), jose 6 (JWT), bcryptjs 3, zod 4, nodemailer 8, stripe 20, playwright 1.58
- **AI:** Vercel AI SDK 4 (`ai`, `@ai-sdk/openai`, `@ai-sdk/react`) — example route at `/api/chat`, example UI at `/chat-example`
- **TypeScript:** 5.9, strict mode

## Commands

```bash
pnpm dev         # dev server on :8497 (Vite + tsx watch)
pnpm build       # Vite client build + esbuild server bundle
pnpm check       # tsc --noEmit
pnpm db:push     # apply drizzle/schema.ts to Postgres
pnpm db:seed     # idempotent seed: tenant + admin@example.com / ChangeMe!2026
```

PostgreSQL setup: `bash setup-postgres.sh` (apt-installs locally, not Docker). Uninstall: `bash setup-postgres.sh uninstall`.

## Request Lifecycle

```
Browser ─fetch→ Vite dev proxy / Express (prod static) ─→ authenticateUser middleware (reads JWT cookie, sets req.user)
 ─→ router (zod.parse body) ─→ requireAuth / requireAdmin if needed ─→ handler
 ─→ server/db.ts helper ─→ drizzle-orm ─→ pg pool ─→ PostgreSQL
 ← JSON { data } or { error }
```

## Auth Tiers

Three levels, applied as Express middleware from `server/middleware/auth.ts`:

| Tier | Middleware | Examples |
|---|---|---|
| Public | none | `POST /api/auth/login`, `POST /api/auth/register` |
| Protected | `requireAuth` | `GET /api/auth/me`, `POST /api/payment/checkout` |
| Admin | `requireAuth` + `requireAdmin` | `POST /api/tenant/invite`, `DELETE /api/tenant` |

Apply at the router level when every handler needs the same tier:

```ts
router.use(requireAuth);
router.put("/users/:id/role", requireAdmin, handler);
```

Never check roles inside a handler when middleware can enforce it.

## Feature Recipe

To add a new feature end-to-end:

1. **Schema (if needed)** — edit `drizzle/schema.ts`, run `pnpm db:push`.
2. **DB helpers** — add async functions to `server/db.ts` using drizzle-orm. Use `.returning()` for writes, destructure `[row]` for single-row reads.
3. **API route** — create `server/routes/<resource>.ts` exporting an Express `Router`. Validate body with zod. Register in `server/routes/index.ts`.
4. **API client method** — add a typed entry to `client/src/lib/api.ts`. Add its query key to `queryKeys` if used with react-query.
5. **Page or component** — create in `client/src/pages/` or `client/src/components/`. Fetch via `useQuery({ queryFn: api.<method> })`. Wire it into `client/src/App.tsx` if it's a page.
6. **Sidebar entry (if it's a page users navigate to)** — add an item to `menuGroups` in `client/src/components/DashboardLayout.tsx`: `{ icon, label: "<i18n_key>", path: "/your-page", roles?: ["admin"] }`. Skip this step if the page is intentionally opt-in by URL only.
7. **i18n** — add every user-facing string as a key in `client/src/locales/de/common.json` AND `en/common.json`. Use `const { t } = useTranslation(); t("key")`. The sidebar `label` is an i18n key, not a literal string.
8. **Verify** — `pnpm check` (types), `pnpm lint`, manual test at `http://localhost:8497`.

## Quick Patterns

Drizzle single-row query:
```ts
const [user] = await db.select().from(users).where(eq(users.id, id)).limit(1);
```

Drizzle insert with returning:
```ts
const [row] = await db.insert(users).values(data).returning();
```

Drizzle transaction:
```ts
await db.transaction(async (tx) => {
  await tx.insert(tenants).values(...);
  await tx.insert(users).values(...);
});
```

Client query:
```ts
const { data } = useQuery({ queryKey: queryKeys.me, queryFn: api.getMe });
```

Client mutation:
```ts
const mutation = useMutation({
  mutationFn: api.login,
  onSuccess: () => toast.success(t("login_success")),
});
```

Express route with validation:
```ts
const schema = z.object({ email: z.string().email(), password: z.string().min(6) });
router.post("/login", async (req, res) => {
  const body = schema.parse(req.body);
  // ...
});
```

## Deployment (sub-path / Halerium)

The template is deployable behind a reverse-proxy sub-path (e.g. Halerium mounts apps at `/apps/<org>/<ws>/<AppName>/`). Three layers are already wired — do not duplicate this logic:

- **Assets** — `vite.config.ts` has `base: "./"` (relative URLs resolved against `document.baseURI`).
- **Client routing** — `client/src/App.tsx` wraps routes in `<Router base={BASE_PATH}>`.
- **API client** — `client/src/lib/api.ts` prepends `BASE_PATH` to `/api/*` calls.

`BASE_PATH` is resolved in `client/src/lib/basePath.ts`: `VITE_BASE_PATH` env → `<base href>` tag → `""`. The server (`server/_core/vite.ts`) injects the `<base href="${prefix}/">` tag at request time, reading `X-Forwarded-Prefix` or the `BASE_PATH` env var.

**Runner bootstrap:** `start.sh` (at repo root) idempotently upgrades Node to v20, installs pnpm + Postgres, handles stale `postmaster.pid` after ephemeral spin-downs, pushes schema, seeds, and launches via package.json scripts only:

- `bash start.sh [PORT]` — prod: builds if needed, runs `pnpm start`.
- `bash start.sh --dev [PORT]` — dev: runs `pnpm dev` (Vite + tsx watch, HMR). Use this while iterating.

Use runner type **`nano`** on Halerium (`standard` silently fails).

Full detail in `DEPLOYMENT.md`.

## How to Add AI Chat to Your App

The template ships with a Vercel AI SDK example at `/chat-example` (requires login, linked from the sidebar as "AI Chat"). Use it as the starting point for any LLM-powered feature.

**Env vars** (add to `.env` — all optional; endpoint returns 503 without a key):

```
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1   # override for OpenAI-compatible endpoints
OPENAI_MODEL=gpt-4o-mini
```

`OPENAI_BASE_URL` lets you swap in Azure OpenAI, OpenRouter, vLLM, or a local Ollama (via its OpenAI-compat shim) without code changes.

**Server — `server/routes/chat.ts`** — `requireAuth` router, zod-validates `{ messages }`, pipes `streamText(...).pipeDataStreamToResponse(res)`. Swap providers by installing `@ai-sdk/anthropic` (or similar) and replacing `openai(model)` with `anthropic(model)`.

**Client — `client/src/components/chat/ChatExample.tsx`** — uses `useChat` from `@ai-sdk/react` with `api: \`${getApiBase()}/chat\`` and `credentials: "include"`. The `getApiBase()` call is required — hard-coding `/api/chat` breaks sub-path deployments.

**To add a new AI feature**: copy `chat.ts`, adjust the system prompt / messages shape, register it in `server/routes/index.ts`, and build a UI using `useChat` (chat) or `useCompletion` (single-turn) pointed at the new endpoint.

## Anti-Patterns

Never:

- Reintroduce tRPC. The API is Express REST by design.
- Add `axios`. Use the `fetch`-based `api.*` wrapper.
- Call `fetch` directly from a component. Go through `client/src/lib/api.ts`.
- Scatter drizzle queries across route handlers. Put them in `server/db.ts`.
- Hardcode `APP_NAME`, support email, or copy strings. Use `shared/config.ts` and i18n keys.
- Assume the app is served from `/`. Use `BASE_PATH` / `getApiBase()` from `client/src/lib/basePath.ts` for client-side URL construction.
- Use `console.log/error/warn` in server code. Import `logger` from `server/utils/logger.ts` and use `logger.info/warn/error/debug`. Pass errors as `logger.error({ err }, "message")`. Logs land in `<pkg-name>_logs/app.log` (all levels) and `error.log` (errors). Dev defaults to `debug`, prod to `info`; override with `LOG_LEVEL`.
- Commit `.env`, `pg-data/`, `uploads/`, `data/`, `dist/`, `*_logs/`.
- Skip git hooks with `--no-verify`.
- Use `any`, `console.log` in committed code, or browser `alert`.

Full list in `CONVENTIONS.md`.
